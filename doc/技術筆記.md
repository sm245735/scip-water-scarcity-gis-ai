## 2026-04-12｜水庫 Comparison API 爬蟲（最重要）

### 背景
需要 2016-2026 的水庫歷史日資料作為 LSTM 訓練目標。嘗試多種方法：

### 失敗記錄

**1. WRA 新版 API（已關閉）**
- URL: `fhy.wra.gov.tw/WraApi/v1/Reservoir/DailyCondition`
- 結果: 404 / Request Rejected

**2. WRA 舊版 Statistics.aspx**
- 發現 CDP 可以操控，但**一次只查一天**
- 沒有 date range 功能，需要手動點擊 365 次
- Excel 下載按鈕綁定當天 postback，價值低

**3. opendata.wra.gov.tw API**
- 即時 API，有 Swagger 文件
- 只保留 1-2 天資料，**不是歷史資料庫**
- 建立每日收集 cron job，逐步建立歷史

**4. WRA 舊版 SOAP wsReservoir.asmx**
- `GetInfos` 方法只有當日水位
- 無歷史查詢功能

### 成功：Comparison API

**找到方式：**
1. 新網站 `fhy.wra.gov.tw/fhyv2/monitor/reservoir` 是一個 Vue 2 應用
2. 藏在 iframe 裡：`disaster/Reservoir/Comparison`（ASP.NET WebForms）
3. 點擊四個分頁時，iframe URL 不變（所以一開始以為沒東西）
4. 從 JS bundle URL 反向追蹤：`/Disaster/bundles/Comparision.js`
5. 在 JS 中找到關鍵程式碼：
```javascript
var n={ST_NO:ReservoirComparisonVM.checkedStations().join(","),
       StartDate:n.StartDate, EndDate:n.EndDate};
$.post(root+"/api/ReservoirHistoryApi/GetComparision", t, ...)
```

**API URL:** `POST https://fhy.wra.gov.tw/Disaster/api/ReservoirHistoryApi/GetComparision`

**請求參數:**
- `ST_NO`: 水庫 ID（最多 9 個，逗號分隔）
- `StartDate`: 開始日期（YYYY/MM/DD）
- `EndDate`: 結束日期（YYYY/MM/DD）

**關鍵限制：** API 只回傳「頭尾」各一筆記錄！
- 查詢 2023/01/01 ~ 2023/12/31 → 只回 2 筆
- 查詢 2023/01/01 ~ 2023/01/01 → 回 1 筆
- **策略：每次區間=1天**

**CDP 操控過程（參考 `cdp_wra.py`）：**
1. WebSocket handshake → CDP protocol
2. `Page.navigate` 導航到目標 URL
3. `Runtime.evaluate` 執行 JavaScript 操作 DOM
4. `Page.captureScreenshot` 截圖確認
5. 複雜互動：選項點擊 → 等 AJAX 回應 → 取結果

**CDP WebSocket 實現（純 Python 標準庫）：**
```python
def ws_handshake(sock, path):
    key = base64.b64encode(os.urandom(16)).decode()
    req = ("GET %s HTTP/1.1\r\nHost: %s:%d\r\n"
            "Upgrade: websocket\r\nConnection: Upgrade\r\n"
            "Sec-WebSocket-Key: %s\r\n"
            "Sec-WebSocket-Version: 13\r\n\r\n")
    sock.sendall(req.encode())
    resp = sock.recv(4096)  # 等待 101 Switching Protocols
```

## 2026-04-12｜TCCIP 降雨資料匯入（進行中）

**目標：** 將 TCCIP 網格化降雨 CSV（新竹市 1960-2023）匯入 `rainfall_grid_data` 表

**問題根本原因：** 兩個獨立的問題
1. DB URL 中密碼含 `@`，需使用 `urllib.parse.quote_plus()` URL encode，否則 `${DB_PASSWORD}@db` 被錯誤解析為 host=`***@db`
2. PostGIS geometry 欄位名稱是 `geom`，但 `gpd.GeoDataFrame(geometry=...)` 預設產生的 geometry 欄位名稱是 `geometry`

**解法：** 
- 使用 `quote_plus(DB_PASS)` 處理密碼
- 使用 `gdf.rename_geometry('geom')` 而非 `rename(columns={'geometry': 'geom'})` 來改 geometry 欄位名稱（`rename_geometry` 會正確更新 GeoDataFrame 的 active geometry column）

**腳本位置：** `/home/sm245735/.openclaw/workspace/workspace/import_rainfall_tccip.py`
**執行環境：** `thesis_python_dev` Docker 容器
**container 內路徑：** `/app/import_rainfall_tccip.py`

**執行紀錄：**
- v1：失敗（Unnamed 欄位 date parse 失敗）→ 用 `usecols=lambda` 過濾
- v2：失敗（`geometry` 改名後 `['geom'] not in index`，因為 `rename(columns)` 不會改 active geometry name）
- v3：失敗（同上）
- v4：**成功** — 使用 `gdf.rename_geometry('geom')` 配合 URL encode 密碼，單一 CSV（2023）成功匯入 53,655 列（147 網格 × 365 天），驗證 `SELECT ST_AsText(geom)` 確認 Point geometry 正確

**成功標準驗證：**
- ✅ 單一 CSV（2023）成功匯入 53,655 列（147 網格 × 365 天）
- ✅ `SELECT ST_AsText(geom) FROM rainfall_grid_data LIMIT 3` 確認 Point(120.87 24.74) 等正確

## 2026-04-12｜Spatial Join 與新竹縣市集水區每日降雨查詢

### 資料現況
| 資料表 | 筆數 | CRS | 說明 |
|--------|------|-----|-------|
| `rainfall_grid_data` | 大量（新竹/台中/台南） | EPSG:4326 | TCCIP 網格降雨點 |
| `reservoir_boundaries` | 129 | EPSG:4326 | 水庫蓄水範圍 |
| `reservoir_catchments` | 839 | EPSG:4326 | 集水區（由 EPSG:3826 轉換）|

### 關鍵 SQL（Spatial Join，學長提供）
```sql
SELECT
    c.basin_name,
    r.data_date,
    AVG(r.rainfall_mm) AS avg_catchment_rainfall
FROM rainfall_grid_data r
JOIN reservoir_catchments c
ON ST_Intersects(r.geom, c.geom)
WHERE r.data_date BETWEEN '2023-01-01' AND '2023-12-31'
GROUP BY c.basin_name, r.data_date
ORDER BY c.basin_name, r.data_date;
```

### 座標系統一致很重要！（學長提醒）
- `rainfall_grid_data`：EPSG:4326（WGS84）
- `reservoir_catchments`：EPSG:3826 → 已轉換成 EPSG:4326
- **若 CRS 不一致，`ST_Intersects` 會算出空結果！**

### 查詢腳本
- **新竹縣市限定版**（先用縣市界線 shapefile 限縮集水區）：`src/gis_analysis/新竹集水區每日降雨查詢_v1_20260412.py`
- 查詢結果已匯出：`data/新竹縣市_集水區每日降雨量_2023.csv`（3285筆，9個集水區）

### 驗證結果
- ✅ 成功查出 2023 年 9 個集水區的每日平均降雨量
- ✅ 中港溪 2023-01-01: 0.0mm, 2023-01-02: 1.4mm（合理）
