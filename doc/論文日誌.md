
```bash

docker exec -it thesis_python_dev bash

# 安裝新套件
pip install 套件名稱

# 更新 requirements.txt
pip freeze > requirements.txt
```

### Q: 資料不見了？
- 確認有使用 Volume：`postgres_data`
- 執行 `docker compose down` 不會刪除 Volume
- 執行 `docker compose down -v` 才會刪除（資料會不見！）

| 2026-04-19 | **§3.2 特徵選擇（論文方法論文字，reviewer 提供）** |
| | |
| | **§3.2 特徵選擇** |
| | |
| | 本研究原規劃使用進水量（inflow）與出水量（outflow）作為 LSTM 輸入特徵，以符合傳統水平衡模型的概念。然而資料品質分析顯示，寶山第二水庫 2016-2023 期間，inflow_cms 與 outflow_cms 僅各有 25 天之有效觀測值（觀測率 0.9%）。此現象反映寶二作為離槽水庫之運作特性——其流率並非每日例行記錄，而是依調度需求不定期登載。 |
| | |
| | 鑑於上述資料限制，本研究改採「降雨–蓄水直接映射」之特徵策略：以每日集水區平均降雨（TCCIP 空間加權）作為輸入側驅動變數，結合當前蓄水量、累積降雨（7 日、30 日）、蓄水量一階差分與日序週期編碼作為 LSTM 輸入，直接預測次日蓄水量。此設計避開了不可靠的中介變數，並在方法論上更接近端對端（end-to-end）的資料驅動預測典範。 |
| | |
| | **Feature 組合（LSTM 實際使用）：** |
| | ```python |
| | FEATURE_COLS = [
| |     "basin_rainfall_tccip_mm",  # 當日降雨
| |     "rainfall_7d_sum",           # 7 天累積降雨
| |     "rainfall_30d_sum",         # 30 天累積降雨
| |     "effective_storage",           # 當前蓄水量
| |     "storage_diff_1d",           # 前日蓄水變化
| |     "doy_sin",                   # 年週期 sin
| |     "doy_cos",                   # 年週期 cos
| | ] |
| | LABEL_COL = "effective_storage"   # shift -1 預測明日蓄水量 |
| | ``` |
| | |
| | **保留但不進 LSTM 的欄位：** |
| | - `inflow_cms` / `outflow_cms`（缺值 97.8%，供論文缺值統計表、口試附錄用） |
| | - `basin_rainfall_self_mm`（水利署單點值，僅供 sanity check） |
