## 水庫日資料

### Comparison API（主要歷史資料）

**目標 URL：** `POST https://fhy.wra.gov.tw/Disaster/api/ReservoirHistoryApi/GetComparision`

**發現方式：** 在 `fhy.wra.gov.tw/fhyv2/monitor/reservoir` 頁面找到 iframe `disaster/Reservoir/Comparison`，從其 JS bundle (`Comparision.js`) 中找到 API endpoint。

**請求格式：**
```
POST https://fhy.wra.gov.tw/Disaster/api/ReservoirHistoryApi/GetComparision
Content-Type: application/x-www-form-urlencoded

ST_NO=10201,10405,...&StartDate=2023/01/01&EndDate=2023/01/01
```

**重要限制：**
- 每次最多 9 個水庫（逗號分隔）
- **API 只回傳「頭尾」各一筆記錄**
- **策略：每次區間=1天**，逐日查詢 = 完整日資料
- 2016-2026 共 10 年，每天 3 次 API call，總計約 10,950 次，6-8 小時跑完

**回傳欄位：**
| 欄位 | 說明 |
|------|------|
| `ST_NO` | 水庫 ID |
| `NAME_C` / `RES_NAME` | 水庫名稱 |
| `DATE` | 日期時間 |
| `WaterLine` | 水位（公尺） |
| `Capacity` | 有效蓄水量（萬噸） |
| `CapacityRate` | 蓄水率 |
| `Inflow_Total` | 總入流量（cms） |
| `Outflow_Total` | 總出流量（cms） |
| `Basin_Rain` | 集水區累積降雨（mm） |
| `UseOutflow` | 發電放水量 |
| `SpillwayOutflow` | 排砂道/溢洪道 |

### SOAP API（即時水庫資料）

**URL：** `https://fhy.wra.gov.tw/ReservoirPage_2011/WS/wsReservoir.asmx/GetInfos`

**發現方式：** WRA 舊版頁面的 `/WS/wsReservoir.asmx?wsdl` WSDL 文件

**請求格式：** SOAP POST（無需認證）

**用途：** 每小時自動收集當日水庫資料，逐步建立歷史資料庫

**回傳欄位：** `ST_NO`, `WaterLine`, `Capacity`, `Inflow`, `Outflow` 等

## 爬蟲經驗摘要

### WRA 網站爬蟲過程（2026-04-12）

| 嘗試 | 目標網站 | 方法 | 結果 | 學到的 |
|------|---------|------|------|--------|
| 1 | `fhy.wra.gov.tw/WraApi/v1/...` | 直接 curl | 404/Request Rejected | API 已經關閉 |
| 2 | `fhy.wra.gov.tw/ReservoirPage_2011/` | CDP socket WebSocket | 部分成功 | ASP.NET PostBack 需要完整 ViewState |
| 3 | `opendata.wra.gov.tw/api/v2/...` | requests.get | ✅ 成功 | 即時 API，但只有1-2天歷史 |
| 4 | `fhy.wra.gov.tw/ReservoirPage_2011/Statistics.aspx` | CDP DOM操控 | 成功但只有月資料 | Statistics.aspx 一次只查一天 |
| 5 | **新網站 `fhyv2/monitor/reservoir`** | CDP iframe分析 → JS bundle | **✅ 成功找到 Comparison API** | Vue 網站藏在 iframe，API 在 JS bundle |
| 6 | `disaster/Reservoir/Comparison` iframe | 從 JS bundle 反向追蹤 | **✅ 找到 `/Disaster/api/ReservoirHistoryApi/GetComparision`** | POST 格式，ST_NO×9限制，只傳頭尾 |

### 關鍵技術

**CDP WebSocket 客戶端（純 Python）：**
- 用 `socket` + `base64` 手刻 WebSocket handshake
- Frame 編碼：opcode=0x81，mask bit=1，extended length 處理
- 收到訊息：解 mask → decode JSON

**新網站結構探索：**
1. `Page.getFrameTree()` 找到 iframe URL
2. 點擊不同 tab 觀察 iframe 是否變化
3. 截圖 `Page.captureScreenshot` + 分析 HTML 定位 UI 元素
4. 從 JS bundle (`Comparision.js`) 找到 API endpoint 和請求格式
